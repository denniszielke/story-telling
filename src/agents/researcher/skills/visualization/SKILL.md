# Functional Architecture Visualization

Use this skill to produce the **Visualization** section of every architecture
proposal. The diagram is a hand-drawn whiteboard sketch generated by DALL·E
from a structured layered description.

## Required inputs

- `title` — short proposal title shown at the top.
- `scenario_description` — one sentence capturing what the diagram depicts.
- `layers` — 3–4 horizontal bands organising the system
  (e.g. *Experience*, *Agents*, *Knowledge & Tools*, *Data & Telemetry*).
- `components` — labelled boxes/doodles inside layers; include any icon hint
  in the label (e.g. *"Search index (vector store)"*).
- `flows` — the key interactions/arrows between components.

## Optional inputs

- `callouts` — 2–4 short value-statements highlighting business or
  architectural value (e.g. *"<5 s p95 retrieval"*).
- `aspect_ratio` — `16:9` (default), `4:3`, or `1:1`.
- `output_path` — where to save the generated PNG.

## Authoring tips

- Reuse the exact component names from the Architecture Concept section so
  the reader can trace the narrative into the picture.
- Prefer 5–9 components total — denser diagrams become unreadable.
- Express each flow as `Source → Target: purpose` for crisp arrows.
- Callouts should reinforce decisions made in the proposal, not introduce
  new ones.

## When to call

Call exactly **once per proposal**, after the Architecture Concept and the
Architecture Decisions have been written. The tool returns the file path of
the saved image; include that path in the final response.
